Primitive Data Types

1.Number

Represents numeric values (integers or floating-point numbers).


2.String

Represents sequences of characters.


3.Boolean

Represents logical values true or false.


5.Null

Represents the intentional absence of any object value


6.Symbol

Represents a unique identifier. Primitive data types are simple, atomic values, and they are stored directly in memory. When you work with primitive data types, you are manipulating the actual value.


Non-Primitive Data Types

1.object

Represents a collection of key-value pairs. Objects are used for more complex data structures and can include functions, arrays, and other objects


2.Array

Represents an ordered list of values.


3.Function

Represents a reusable block of code. Non-primitive data types are more complex, and when you work with them, you are dealing with a reference to the actual value stored in memory. This means that when you assign a non-primitive variable to another, you are working with references, not the actual values.
